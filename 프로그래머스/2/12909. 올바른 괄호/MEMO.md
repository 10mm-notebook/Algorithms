# [Programmers] Level 2 - 올바른 괄호 (Python)

## 1. 문제 정보


**날짜:** 2026. 02. 16 



**출처:** 프로그래머스 - 올바른 괄호 (Level 2) 


**핵심 알고리즘:** 스택 (Stack) 



## 2. 문제 설명

주어진 문자열이 올바른 괄호 형식인지 판단하는 문제입니다. 

`"()()"` 또는 `"(())()"` 는 올바른 괄호입니다.
`")()("` 또는 `"(()("` 는 올바르지 않은 괄호입니다. 



## 3. 접근 방식 및 사고 과정

괄호의 짝이 맞는지 확인하기 위해 **선입후출(LIFO)** 구조인 **스택(Stack)**을 활용하였습니다. 

1. **열린 괄호 `'('` 처리:** 문자열을 순회하며 `'('`를 만나면 스택에 추가(`append`)합니다. 


2. **닫힌 괄호 `')'` 처리:** `')'`를 만나면 스택에서 하나를 꺼냅니다(`pop`). 
이때, 스택이 비어있는 상태에서 `')'`가 들어오면 짝이 맞지 않으므로 즉시 `False`를 반환합니다. 




3. **마지막 검증:** 모든 문자열을 순회한 후 스택의 길이를 확인합니다. 


스택이 비어있다면(`len == 0`) 모든 짝이 맞은 것이므로 `True`. 


스택에 요소가 남아있다면 열린 괄호가 더 많은 것이므로 `False`. 





## 4. 주요 케이스 분석 (Troubleshooting)

메모에서 정리한 세 가지 핵심 상황입니다:

**케이스 1 (정상):** `(` 수와 `)` 수가 일치하여 최종적으로 스택이 비게 됨  `True`. 


**케이스 2 (닫힘 부족):** `(` 수에 비해 `)` 가 부족하여 마지막 검증 때 스택이 남아 있음  `False`. 


**케이스 3 (열림 부족/순서 오류):** 스택이 빈칸인데 `)`가 먼저 오거나 더 많이 들어옴  `False`. 



## 5. 최종 코드

```python
def solution(s):
    s = list(s)
    stack = []
    
    condition = True
    
    for i in range(len(s)):
        if s[i]=="(":
            stack.append("(")
        else:
            if len(stack)<=0:
                condition = False
            else:
                if len(stack)>=1:
                    del stack[-1]
                else:
                    condition = False
    
    if len(stack)>=1:
        condition = False         
    
    return condition

```

## 6. 복잡도 분석

**시간 복잡도: **
문자열 `s`의 길이만큼 딱 한 번만 순회하므로 입력 크기에 비례하는 선형 시간이 소요됩니다.


**공간 복잡도: **
최악의 경우(예: 모든 문자가 `'('`인 경우) 스택에 모든 문자를 담아야 하므로 문자열 길이에 비례하는 공간이 필요합니다.



## 7. 공부한 점 및 반성

스택의 `pop`과 `append`를 통해 괄호의 쌍을 효율적으로 제어하는 법을 익혔습니다. 

`if len(stack) == 0` 조건을 통해 예외 상황(빈 스택에 닫힌 괄호가 오는 경우)을 조기에 차단하는 로직의 중요성을 깨달았습니다. 

괄호 문제는 스택의 가장 전형적이면서도 강력한 응용 사례임을 확인했습니다.
